## 一、 功能定位
面向电子制造产线测试工程师，提供对产线各工序测试日志（如 Function Test、Radio Test、Txpower、Firmware Install、TPM Verify、OBA 等）的自动化解析、结构化报告生成与异常智能诊断能力。

---

## 二、 功能模块

### 模块 1：日志解析引擎（规则驱动）
* **日志类型自动识别**：根据文件名模式（如 `*_Function_Test.txt`）或内容特征（如包含 `SHMOO` 关键字）自动匹配对应的解析器。
* **关键信息提取**：
  * 设备标识：MAC 地址、序列号（SN）、SMT 编号、生产日期
  * 测试结果：各工序 PASS/FAIL 状态、测试耗时
  * 关键参数：DDR 时序参数、TX Power 校准值、固件版本、NAND Flash 信息
  * 异常记录：ERROR、WARNING、Failed 等关键字及上下文
* **结构化输出**：统一 JSON Schema，支持前端可视化渲染。

### 模块 2：Shmoo 图可视化
* **ASCII 字符解析**：解析日志中的 Shmoo 图（`+`、`-`、`X`、`S` 字符），转换为可视化热力图。
* **最佳采样点标记**：自动标记 `X` 位置（最佳采样点），绘制安全裕度边界。
* **跨设备对比**：支持多台设备 Shmoo 图叠加对比，检测信号完整性退化趋势。

### 模块 3：异常智能诊断（可选 AI 增强）
* **异常自动分类**：将异常分为"正常"、"警告"、"严重"三级。
* **自然语言摘要**：一键生成测试报告摘要（如"本设备共完成 6 道工序，全部通过。发现 3 个低级别警告，均不影响出货。"）。
* **相似案例检索**：在历史日志库中检索相似异常，辅助根因分析。
* **根因深度分析**：结合历史数据推测可能原因（需云端大模型支持）。

### 模块 4：跨设备趋势分析
* **批次对比**：同一工序在不同批次间的通过率对比。
* **异常趋势**：某类异常在最近 N 台设备中的出现频率趋势。
* **与 CPK 数据联动**：日志中提取的校准参数（如 TX Power）与最终 CPK 的相关性分析。

## 四、 AI 模型配置（Settings 新增页签）

在设置面板中新增 **"AI 模型"** 页签，支持用户自行配置模型服务：

### 4.1 云端模型配置
* **服务商选择**：OpenAI、Anthropic、Google、通义千问、DeepSeek 等
* **API Key**：用户自行填写
* **Base URL**：支持自定义（兼容 OpenAI 格式的第三方服务）
* **模型名称**：如 `gpt-4o`、`claude-sonnet-4-20250514`、`qwen-plus` 等
* **连接测试**：一键测试 API 连通性

### 4.2 本地/内网模型配置
* **Ollama 服务地址**：如 `http://localhost:11434` 或 `http://192.168.1.100:11434`
* **模型名称**：如 `qwen2.5:1.5b`、`llama3.2:3b`、`phi3:mini`
* **连接测试**：一键测试 Ollama 连通性
* **推荐模型列表**：内置推荐的小模型清单（按任务类型分类）

### 4.3 模型使用策略
* **异常分类模型**：用于自动判断异常严重程度
* **摘要生成模型**：用于生成自然语言测试报告
* **根因分析模型**：用于深度异常诊断（可选）
* **离线优先**：优先使用本地模型，无本地模型时回退到云端

### 4.4 隐私与安全
* **API Key 加密存储**：使用 Tauri 安全存储机制
* **数据脱敏**：上传云端前自动脱敏设备标识（MAC、SN）
* **使用日志**：记录模型调用历史，便于审计

## 六、 测试与验收标准

### 1. 产线日志分析
* **解析准确性验收**：
  * 成功解析 6 种日志类型（Function Test、Radio Test、Txpower、Install_APboot、Verify_TPM_AOS、OBA Test）。
  * 设备标识（MAC、SN、SMT 编号）提取准确率 100%。
  * 工序 PASS/FAIL 状态提取准确率 100%。
  * 异常记录提取无遗漏。
* **可视化验收**：
  * Shmoo 图正确渲染为热力图，最佳采样点标记准确。
  * 工序结果表格清晰展示各工序状态、耗时、警告数。
  * 异常列表支持按严重程度筛选。
* **AI 功能验收**（如启用）：
  * 异常分类准确率 > 90%（与人工标注对比）。
  * 自然语言摘要覆盖所有关键信息（设备信息、工序结果、异常统计）。
  * 模型调用响应时间 < 10 秒（云端）/ < 5 秒（本地）。
* **性能验收**：
  * 导入包含 6 个日志文件的文件夹，解析完成时间 < 3 秒。
  * 历史数据库查询响应时间 < 500ms。
  * 批次对比图表渲染时间 < 1 秒。

### 2. AI 模型配置验收
* **云端模型配置**：
  * 支持 OpenAI 兼容 API 格式的服务商配置。
  * API Key 加密存储，不暴露于日志或网络请求中。
  * 连接测试功能正常，能正确反馈连通状态。
* **本地模型配置**：
  * 支持 Ollama 服务地址配置。
  * 推荐模型列表展示正确。
  * 本地模型不可用时正确回退到云端或降级模式。
* **隐私保护**：
  * 上传云端前自动脱敏设备标识。
  * 模型调用审计日志完整记录。
